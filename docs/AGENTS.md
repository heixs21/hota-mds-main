# Agent Rules

## Requirement clarification
Before implementing any feature from PRD:
1. Read PRD.md first
2. Identify missing or ambiguous requirements
3. Ask the user focused clarification questions before coding
4. Do not start implementation until the key ambiguities are resolved

## Project memory
After clarification, generate and maintain:
- docs/SPEC.md
- docs/PLAN.md
- docs/STATUS.md
- docs/HANDOFF.md
- docs/DECISIONS.md

## Implementation discipline
- Only implement one milestone at a time
- Update STATUS.md and HANDOFF.md after each development round
- Do not change architecture without recording the reason
