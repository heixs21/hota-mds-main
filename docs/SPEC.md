# 和泰智造数屏系统 SPEC

## 1. 目标

建设面向外部客户参观的数字化工厂展示大屏系统，通过左右双屏展示工厂设备运行、生产进度、排产计划、能耗概况等核心信息，提升客户参观和领导接待时的数字化展示效果。

一期前段核心目标：

- 支持左右双屏独立 URL 展示。
- 支持全屏展示与自动轮播。
- 真实接入设备、产量、排产、能耗数据。
- 支持后台配置展示内容、轮播策略、台账、编码映射、阈值、数据源。
- 支持后台查看数据源健康状态。
- 数据源异常时，大屏继续显示最近一次成功数据。

## 2. 非目标

一期前段不包含：

- 内部用户 Web 报表。
- 报修系统真实接入与报修数据展示。
- 3D 仿真真实开发与联动。
- 复杂多角色权限体系。
- 前端直接访问 SAP、排产库、能耗库、OPCUA 等外部系统。

说明：

- 内部 Web 报表作为二期项目。
- 报修系统接入作为一期后段项目。
- 3D 仿真作为一期后段低优先级功能，优先级低于报修系统接入。

## 3. 用户角色

### 3.1 外部参观用户

客户、来访人员、领导接待对象。

使用方式：

- 仅观看大屏。
- 不需要登录。
- 不进行交互操作。

### 3.2 后台管理员

一期不做复杂角色拆分，后台默认以管理员账号为主。

管理员能力：

- 登录后台。
- 配置欢迎语、Logo、宣传图片。
- 配置左右屏页面内容、轮播顺序、轮播时长。
- 配置页面模块开关。
- 维护设备、产线、区域、订单、物料等台账。
- 维护本系统主编码与外部系统编码映射。
- 配置数据源连接信息。
- 配置指标阈值、延期预测参数、单日有效工作时间。
- 查看数据源健康状态与最近更新时间。

## 4. 核心业务流程

### 4.1 大屏展示流程

1. 运维人员打开左屏 URL 和右屏 URL。
2. 左右屏分别进入全屏展示模式。
3. 系统按后台配置自动轮播页面。
4. 前端从本系统后端 API 获取缓存后的展示数据。
5. 若数据源正常，展示最新数据。
6. 若数据源异常，大屏继续展示最近一次成功数据，不在大屏提示数据过期。
7. 后台健康状态页显示数据源异常、最近成功更新时间和过期状态。

建议 URL：

- `/screen/left`
- `/screen/right`

### 4.2 数据接入流程

1. 管理员在后台配置外部数据源连接信息。
2. 后端定时任务按配置周期拉取数据。
3. 采集或适配层将外部数据转换为本系统标准数据模型。
4. 后端写入缓存表或缓存层。
5. 大屏接口读取缓存数据并返回给前端。
6. 后台数据源健康页记录每个数据源的连接状态、最近成功时间和错误信息。

默认刷新周期：

- 设备：5 分钟，可配置。
- 产量：5 分钟，可配置。
- 排产：5 分钟，可配置。
- 能耗：5 分钟，可配置。

### 4.3 排产展示流程

1. 后端从排产系统读取未完工订单。
2. 系统根据本系统主编码进行产线、订单、物料映射。
3. 甘特图按产线分组展示。
4. X 轴以天为最小单位。
5. 默认展示从当天起未来 30 日，可后台配置。
6. 超出 30 日窗口的订单仅截取可见部分展示。
7. 订单过多时自动纵向滚动。
8. 未开工订单仅显示计划，不做延期判断。
9. 已延期订单整单标红。
10. 暂停订单使用特殊颜色显示。

### 4.4 延期预测流程

1. 获取订单目标产量。
2. 获取当前已产数量。
3. 计算剩余数量。
4. 根据优先级获取有效产能。
5. 计算预计完工时间。
6. 与计划完工时间对比。
7. 输出正常、风险、延期等状态。

计算逻辑：

```text
剩余数量 = 订单目标产量 - 当前已产数量
预计完工时间 = 当前时间 + 剩余数量 / 有效产能
```

有效产能优先级：

1. 最近 2 小时实际平均产量。
2. 最近 1 个班次平均产量。
3. 该产线该物料历史平均产量。
4. 后台配置的标准产能。

说明：

- 不考虑班次、非工作时间、计划停机、换型时间。
- 仅支持后台配置单日有效工作时间。

## 5. 关键页面

### 5.1 左屏大屏页面

用途：综合运行展示。

模块：

- Logo、欢迎语、当前时间。
- 设备运行概览。
- 产量执行概览。
- 近 8 小时产量趋势。
- 区域能耗概览。
- 报修概览占位区。

说明：

- 报修系统未接入前，报修区域显示占位内容。
- 报修不作为一期前段验收阻塞项。

### 5.2 右屏大屏页面

用途：生产动态展示。

模块：

- 未完工订单排产甘特图。
- 延期风险与状态说明。
- 右下角 3D 设备/产线仿真预留区。

说明：

- 3D 仿真作为一期后段低优先级功能。
- 一期前段仅保留占位区即可。

### 5.3 后台管理页面

模块：

- 登录页。
- 展示内容配置。
- 左右屏轮播配置。
- 页面模块开关。
- 台账维护。
- 编码映射维护。
- 数据源配置。
- 指标阈值与预测参数配置。
- 数据源健康状态页。
- 操作日志。

## 6. 数据对象

### 6.1 主数据对象

- 设备
- 产线
- 区域
- 订单
- 物料

### 6.2 映射对象

本系统自建主数据编码，并映射其他系统编码。

映射范围：

- 设备编码映射。
- 产线编码映射。
- 订单号映射。
- 物料号映射。
- 区域编码映射。

### 6.3 展示数据对象

- 设备状态快照。
- 产量统计快照。
- 近 8 小时产量趋势。
- 未完工订单排产数据。
- 延期预测结果。
- 区域能耗快照。
- 报修占位或后段报修统计。
- 数据源健康状态。

### 6.4 数据源对象

- OPCUA / Modbus TCP：设备与机械手数据。
- SAP RFC：订单与物料信息。
- 排产系统数据库：未完工订单、产线、计划开始/结束时间。
- 能耗数据库：区域能耗数据。
- WMS：预留。
- 报修系统：一期后段。

## 7. 权限模型

一期不做复杂角色体系。

默认权限：

- 大屏访问：不登录，仅用于展示。
- 后台访问：需要登录。
- 管理员：拥有全部后台配置权限。

安全要求：

- 外部数据源连接信息只能在后台配置。
- 外部数据源连接信息不能写死在代码中。
- 敏感连接信息建议加密存储或通过环境密钥保护。
- 后台关键操作记录操作日志。

## 8. 验收标准

一期前段验收：

- 大屏可在双屏场景稳定全屏展示。
- 左右屏分别通过独立 URL 打开。
- 支持默认 60 秒自动轮播，并可后台配置。
- 设备、产量、排产、能耗数据必须真实接入。
- 可展示设备总数、运行数、异常数、状态占比。
- 可展示各产线当前订单、目标产量、已产数量、完成率、近 8 小时产量。
- 排产模块可按产线分组展示未来 30 日内未完工订单，并支持自动滚动。
- 可对已开工未完工订单进行延期风险判断。
- 可按区域展示能耗概况。
- 后台可维护台账、编码映射、展示内容、轮播方式、关键阈值、数据源配置。
- 后台可查看数据源健康状态。
- 数据源异常时，大屏显示最近一次成功数据，不白屏。

不作为一期前段验收阻塞项：

- 报修模块。
- 3D 仿真。
- 内部 Web 报表。

## 9. 技术约束

- 后端：Django + Django REST Framework。
- 前端：React。
- 数据库：MySQL，独立部署。
- 系统部署：Ubuntu + Docker。
- 容器拆分：backend、frontend/nginx、collector。
- OPCUA 采集：建议独立 collector 服务。
- 前端不直接访问外部系统。
- 后端定时拉取并缓存外部数据。
- 默认刷新周期 5 分钟，后续可后台配置。
- 数据库历史数据永久保留。
