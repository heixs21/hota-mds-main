# 和泰智造数屏系统 PRD

## 1. 项目定位

本项目建设目标为面向外部客户参观的数字化工厂展示大屏，重点体现工厂设备运行、生产进度、排产计划、能耗概况和产线动态仿真能力，提升客户参观时的企业形象与数字化展示效果。

内部用户在 Web 端查看各类生产、设备、能耗、报修等报表的需求暂未明确，不纳入一期大屏建设范围，作为二期项目另行规划。

## 2. 建设范围

### 2.1 一期范围：外部参观大屏

一期聚焦拼接屏展示与参观场景，主要包含：

- 欢迎展示：公司 Logo、欢迎语、宣传图片、全屏展示与自动轮播。
- 设备信息：设备总数、运行数、异常数、状态占比图。
- 产量统计：各产线当前订单、目标产量、已产数量、完成率、近 8 小时产量柱状图。
- 交付预测：基于剩余数量、有效产能与计划完工时间判断是否存在延期风险。
- 排产展示：按产线分组展示所有未完工订单，X 轴以天为单位，最多显示未来 30 日。
- 3D 仿真：以三维设备或产线模型方式展示生产流程，作为一期后段低优先级功能实施，优先级低于报修系统接入与报修相关展示。
- 能耗统计：按区域展示能耗概况，体现能源管理能力。
- 后台配置：欢迎语、图片、Logo、轮播方式、页面显示开关、台账、编码映射、阈值参数等。

### 2.2 一期后段执行：报修相关内容

由于报修系统取数存在困难，报修取数与报修数据展示不作为一期前段核心交付内容，建议放在一期项目后段执行。

后段可视报修系统接口条件逐步接入：

- 今日报修数
- 处理中报修数
- 已关闭报修数
- 平均响应时长
- TOP 故障设备
- 最新报修列表

如报修系统接口或数据库暂时无法稳定接入，则大屏中保留报修展示区域或配置开关，不影响一期核心大屏上线。

### 2.3 二期范围：内部 Web 报表

内部用户在 Web 端查看各种报表的需求作为二期项目处理，具体报表内容、权限、交互方式和统计口径待后续调研确认。

二期可预留方向包括：

- 生产报表
- 设备报表
- 能耗报表
- 排产报表
- 报修报表
- 异常分析与历史趋势
- 用户权限与报表导出

## 3. 展示方案

### 3.1 展示场景

- 使用场景：客户参观、领导接待、对外展示。
- 展示设备：拼接屏。
- 展示语言：中文。
- 展示方式：全屏自动轮播。
- 轮播节奏：默认每页 60 秒，可后台配置。

### 3.2 双屏布局

采用左右双屏分别展示的方式。

左屏建议作为综合运行展示：

- Logo、欢迎语、当前时间
- 设备运行概览
- 产量执行概览
- 近 8 小时产量趋势
- 区域能耗概览
- 报修概览预留区或后段接入区

右屏建议作为生产动态展示：

- 未完工订单排产甘特图
- 延期风险与状态说明
- 右下角 3D 设备/产线仿真展示区

## 4. 核心业务规则

### 4.1 设备状态

- 设备总数来自设备台账。
- 状态为“运行”的设备计入运行数。
- 除“运行”外，其余状态均计入异常数，包括停机、报警、离线等。

### 4.2 产线与订单

- 一条产线同一时刻只生产一个订单。
- 一个订单同一时刻只分配到一条产线（待定）。

### 4.3 排产甘特图

- 展示对象：所有状态不是“完工”的生产订单。
- 展示维度：按产线分组。
- 时间粒度：以天为最小单位。
- 时间范围：从当天起向后展示，最多未来 30 日，天数可后台配置。
- 超出未来 30 日窗口的订单，只截取可见部分展示。
- 订单过多时自动纵向滚动。
- 未开工订单显示计划排产，但不做延期判断。
- 已延期订单整单标红。
- 暂停订单使用特殊颜色显示。

### 4.4 延期预测

延期预测建议采用“剩余产量 / 有效产能”推算预计完工时间，再与计划完工时间对比。

计算逻辑：

- 剩余数量 = 订单目标产量 - 当前已产数量
- 预计完工时间 = 当前时间 + 剩余数量 / 有效产能
- 如果预计完工时间晚于计划完工时间，则判定为延期或延期风险

有效产能建议按以下优先级取值：

- 最近 2 小时实际平均产量
- 最近 1 个班次平均产量
- 该产线该物料历史平均产量
- 后台配置的标准产能

## 5. 数据来源

| 数据内容 | 来源 | 接入方式 | 说明 |
| --- | --- | --- | --- |
| 设备与机械手数据 | 机台、机械手 | OPCUA / Modbus TCP | 设备状态、关键点位、抓取状态等 |
| 订单与物料信息 | SAP | RFC | 订单号、物料号、计划信息等 |
| 排产计划 | 排产系统 | 数据库 | 未完工订单、产线、计划开始/结束时间 |
| 能耗数据 | 能耗电脑 | 数据库 | 按区域统计能耗 |
| 物料补充信息 | WMS | 待确认 | 作为补充数据源预留 |
| 报修数据 | 报修系统 | 待确认 | 一期后段接入，前段不作为核心依赖 |

各系统暂无统一编码，因此一期需建设编码映射能力，至少覆盖设备、产线、订单、物料、区域等关键对象。

## 6. 后台管理

一期后台管理需支持：

- 欢迎语、Logo、宣传图片配置
- 左右屏页面内容、轮播顺序、轮播时长配置
- 页面模块显示开关
- 设备台账、产线台账、区域台账维护
- 数据源连接配置
- 编码映射维护
- 指标阈值与延期预测参数配置
- 甘特图展示窗口、状态颜色、自动滚动策略配置
- 用户账号、角色权限与操作日志

## 7. 技术与部署

- 后端：Django + Django REST Framework
- 前端：React
- 数据库：MySQL
- 部署环境：内网服务器
- OPCUA：建议独立采集服务部署
- 页面要求：支持全屏展示，数据异常时不白屏，显示最近更新时间
- 历史数据：建议保留 1 年

## 8. 实施优先级

建议一期按以下顺序推进：

1. 完成设备、产线、区域、订单、物料等基础台账及编码映射梳理。
2. 完成设备、产量、排产、能耗等核心数据接入。
3. 完成双屏大屏展示、全屏播放与自动轮播。
4. 完成后台配置能力与基础运维能力。
5. 在报修系统取数条件具备后，补充报修数据接入与展示。
6. 在一期后段进一步实施 3D 仿真展示，且优先级低于报修系统接入与报修相关功能。

## 9. 主要风险

- 多系统编码不统一，需提前梳理映射关系。
- 报修系统取数存在困难，需作为一期后段执行，避免影响核心上线。
- 内部 Web 报表需求尚未明确，需作为二期单独调研和规划。
- 数据源刷新频率不一致，需通过后端聚合与缓存保证大屏展示稳定。

## 10. 验收标准

- 大屏可在双屏场景稳定全屏展示。
- 支持默认 60 秒自动轮播，并可后台配置。
- 可展示设备、产量、排产、能耗等一期核心模块。
- 排产模块可按产线分组展示未来 30 日内未完工订单，并支持自动滚动。
- 可对已开工未完工订单进行延期风险判断。
- 后台可维护台账、编码映射、展示内容、轮播方式和关键阈值。
- 报修模块不作为一期前段核心验收阻塞项。
- 3D 仿真作为一期后段低优先级功能，不作为一期前段核心验收阻塞项。
